⚙️ Lab Deployment Steps

  1. Ensure Kind, kubectl, Docker & Python 3 are installed.
  2. Run the deploy script from the root directory:
python3 deploy.py
💡 Note: This script sets up a Kubernetes cluster using kind, applies manifests, and exposes the vulnerable app on http://localhost:8080 or NodePort 30080.

🧠 What’s Included?

  • ⚠️ Web Application Vulnerabilities: Exploitable OWASP-10 vulnerabilities like SSRF, RCE, SSTI, Git Secrets Leakage, and metadata exposure—
    mirroring real-world misconfigured production apps.
  • 🧠 Post-Exploitation Arsenal: Simulated privilege escalation via container breakout, token theft, and secret harvesting.
    Includes lateral movement through compromised service accounts and hidden backdoors via DaemonSets.
  • 🧩 Advanced Threat Chain Simulation: Fully aligned with the MITRE ATT&CK framework and Cyber Kill Chain—from Initial Access to Actions on Objectives—
    with mapped Kubernetes-specific TTPs.
  • 🔍 Detection & Threat Hunting: Real-time telemetry via Falco (rules for crypto miners, reverse shells, and suspicious binaries),
    Loki (forensic logs), and Grafana dashboards for visualizing attack footprints.
